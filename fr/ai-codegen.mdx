## Flux de travail recommandé

1. Ouvrez la page de l'API cible et cliquez sur « Copier la page » en haut à droite ou dans les actions contextuelles.
2. Envoyez le contenu complet de la page copiée directement à GPT, Claude ou tout autre modèle de génération de code.
3. Précisez simplement votre langage cible, les paramètres métier et le traitement des réponses souhaité ; il n'est pas nécessaire de restructurer manuellement les chemins d'accès, le corps de la requête ou la structure de la réponse.
4. Après avoir intégré le code généré dans votre projet local, revenez à la documentation pour vérifier le Base URL, la méthode d'authentification et les champs obligatoires.

## Conseils de prompt

> Le contenu ci-dessous est le contenu intégral de la page, copié directement depuis la page de documentation API.
> Veuillez générer du code d’appel en Python/Node.js/Go à partir de ce contenu.
> Exigences :
> 1. Utiliser l’authentification Bearer Token
> 2. Utiliser https://api.aiid.edu.kg/v1 pour Base URL
> 3. Conserver le corps de requête complet et la gestion des erreurs
> 4. Indiquer dans les commentaires les éléments à remplacer : model, clé API et paramètres métier
> 5. Si cette interface est une interface de tâche asynchrone, ajouter un exemple de requête pour consulter le résultat de la tâche

## Vérification avant livraison

- Le Base URL correct est-il renseigné ?
- Le chemin de l'interface utilisé est-il correct ?
- Le `Authorization: Bearer YOUR_API_KEY` est-il inclus ?
- Les champs obligatoires ont-ils été transmis conformément à la documentation ?
