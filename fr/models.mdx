## Groupement de modèles

### Nano Banana

Principalement utilisé pour la génération et l'édition d'images, supporte également l'accès au format natif Gemini.



| Modèle | Méthode d'accès principale |

|---|---|

| `nano-banana` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-2` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-2-2k` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-2-4k` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-pro` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-pro-2k` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |

| `nano-banana-pro-4k` | `/v1/images/generations` / `/v1/images/edits` / `/v1beta/...` |



### Sora

### Kling Images

Fournit publiquement, via `/v1/images/generations`, des capacités de génération d’images Kling, d’images Omni et d’essayage virtuel.



| Modèle | Méthode d'accès principale |

|---|---|

| `kling-image` | `/v1/images/generations` |

| `kling-omni-image` | `/v1/images/generations` |

| `kolors-virtual-try-on-v1` | `/v1/images/generations` |

| `kolors-virtual-try-on-v1-5` | `/v1/images/generations` |



### Kling Videos

L'interface dédiée au modèle et l'entrée vidéo unifiée sont toutes deux publiques.



| Modèle | Méthode d'accès principale |

|---|---|

| `kling-video` | `/kling/v1/...` / `/v1/videos` |

| `kling-*` | `/kling/v1/...` / `/v1/videos` |



### Hailuo

Utilise l'accès par capacité vidéo unifiée, adapté aux scénarios de génération de vidéo à partir de texte (Text-to-Video).



| Modèle | Méthode d'accès principale |

|---|---|

| `hailuo*` | `/v1/videos` |



### Vidu

Se concentre sur la description de la capacité vidéo unifiée, adapté aux scénarios d'image unique, de première/dernière image et de multi-images de référence.



| Modèle | Méthode d'accès principale |

|---|---|

| `vidu*` | `/v1/videos` |
