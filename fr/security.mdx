## Authentification (Authentication)

API utilise le mécanisme standard Bearer Token pour l'authentification.

### Format du Header
```text
Authorization: Bearer YOUR_API_KEY
```

## Spécifications de sécurité des jetons

### 1. Principe du moindre privilège
Si vous distribuez une clé API à des partenaires externes, veuillez restreindre les éléments suivants pour cette clé dans la console :
- **Portée du modèle** : autoriser uniquement l'appel de modèles spécifiques.
- **Délai d'expiration** : définir une expiration à court terme pour réduire les risques.
- **Liste blanche d'IP** : restreindre les adresses IP des appelants.

### 2. Interdiction stricte du codage en dur côté client
N'inscrivez jamais directement le jeton `sk-` dans le code JavaScript frontal ou dans une application mobile. Tous les appels API doivent transiter par un serveur backend ou être générés via notre système de distribution sécurisé sous forme de jetons à court terme.

### 3. Confidentialité des données
Nous ne stockons pas le contenu de vos conversations. API agit uniquement en tant que nœud de transit haute performance ; le contenu des conversations circule en mémoire et est transmis directement au fournisseur de services sous-jacent.

## Surveillance et audit

Vous pouvez surveiller en temps réel les détails de chaque appel API sur la page « Logs » de la console. Pour toute question relative à la sécurité, veuillez envoyer un e-mail à support@aiid.edu.kg.
