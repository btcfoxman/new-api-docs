## Инструкция по использованию

1. Откройте страницу целевого API и нажмите `试一试`.
2. Укажите `api_key` на странице. Это значение сохраняется только локально в браузере, не синхронизируется с сервером и не передается другим участникам команды.
3. Заполните параметры пути (path), параметры запроса (query) или тело запроса (body) в соответствии с требованиями API.
4. Нажмите `发送`, чтобы просмотреть результат ответа и сообщения об ошибках.

## Рекомендации по отладке

- Сначала проверьте правильность Base URL, пути API и названия модели.
- Для API с длительными задачами сначала сохраните возвращенный `task_id` или `operation_name`, а затем вызовите интерфейс запроса статуса.
- Если API поддерживает несколько режимов параметров, используйте пример тела запроса из документации для проверки минимально необходимого набора параметров.

## Использование совместно с кодом, сгенерированным ИИ

Рекомендуется сначала выполнить проверку минимально рабочего запроса на странице документации, затем нажать «Копировать страницу» и отправить все содержимое ИИ для генерации финального кода вызова.
Так AI получает содержимое страниц API, уже проверенное человеком, поэтому в пути, способе аутентификации и схеме параметров с меньшей вероятностью возникнут ошибки.
