## Проверка токена (Authentication)

API использует стандартный механизм Bearer Token для аутентификации.

### Формат Header
```text
Authorization: Bearer YOUR_API_KEY
```

## Правила безопасности токенов

### 1. Принцип наименьших привилегий
Если вы передаете ключ API внешним партнерам, ограничьте в консоли для этого ключа следующие параметры:
- **Доступные модели**: разрешите вызов только определенных моделей.
- **Срок действия**: установите короткий срок действия для снижения рисков.
- **Белый список IP**: ограничьте IP-адреса вызывающей стороны.

### 2. Запрет на жесткое кодирование в клиенте
Никогда не прописывайте токен `sk-` напрямую в коде фронтенда на JavaScript или в мобильном приложении. Все вызовы API должны проходить через серверную часть (бэкенд) или использовать краткосрочные токены, созданные через нашу систему безопасного распределения.

### 3. Конфиденциальность данных
Мы не храним содержание ваших диалогов. API выступает исключительно как высокопроизводительный промежуточный узел: содержимое диалогов обрабатывается в оперативной памяти и напрямую передается базовому поставщику услуг.

## Мониторинг и аудит

Вы можете в реальном времени отслеживать детали каждого вызова API на странице «Журналы» в консоли. Если у вас возникли вопросы по безопасности, отправьте письмо по адресу support@aiid.edu.kg.
