## 使用方式

1. 開啟目標介面頁並點擊 `试一试`
2. 填寫 `api_key`
3. 按 API 要求填寫路徑參數、查詢參數或請求主體
4. 發送請求並查看回傳結果

## 除錯重點

- 先驗證最小可用請求
- 長任務 API 優先確認建立回傳中的 `task_id` 或 `operation_name`
- 檔案上傳介面必須確認 `multipart/form-data` 和檔案欄位名稱正確
- 如果失敗，先看 HTTP 狀態碼、`error.code`、`error.message`

## 與 AI 生成程式碼配合

建議先在當前 API 頁面驗證一次最小可用請求，再使用「複製頁面」將整頁內容傳送給 AI 生成程式碼。

這樣能先確認：

- 路徑和鑑權方式正確
- 當前參數模式可執行
- 非同步任務 API 需要儲存什麼查詢識別碼
