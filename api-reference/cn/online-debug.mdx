## 使用方式

1. 打开目标接口页并点击 `试一试`
2. 填写 `api_key`
3. 按接口要求填写路径参数、查询参数或请求体
4. 发送请求并查看返回结果

## 调试重点

- 先验证最小可用请求
- 长任务接口优先确认创建返回中的 `task_id` 或 `operation_name`
- 文件上传接口必须确认 `multipart/form-data` 和文件字段名正确
- 如果失败，先看 HTTP 状态码、`error.code`、`error.message`

## 与 AI 生成代码配合

建议先在当前接口页验证一次最小可用请求，再使用“复制页面”把整页内容发给 AI 生成代码。

这样能先确认：

- 路径和鉴权方式正确
- 当前参数模式可执行
- 异步任务接口需要保存什么查询标识
