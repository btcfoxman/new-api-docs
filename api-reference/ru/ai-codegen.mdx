## Рекомендуемый подход

1. Откройте страницу нужного API и нажмите «Копировать страницу».
2. Отправьте скопированное содержимое страницы целиком в AI.
3. Укажите AI используемый язык программирования, бизнес-параметры и способ обработки ответа.
4. Попросите AI сгенерировать полный код запроса, обработку ошибок и логику опроса асинхронных задач (если API предполагает работу с задачами).
5. Вернитесь к документации или странице тестирования, чтобы проверить соответствие имен полей, путей и методов аутентификации.

## Рекомендации по промптам

> Ниже приведено полное содержимое страницы, которое я напрямую скопировал со страницы документации API.
> Пожалуйста, сгенерируйте исполняемый код вызова на основе этих материалов.
> Требования:
> 1. Использовать аутентификацию Bearer Token
> 2. Для Base URL использовать https://api.aiid.edu.kg
> 3. Сохранить полное тело запроса и обработку ошибок
> 4. В комментариях отметить model, API Key и бизнес-параметры, которые необходимо заменить
> 5. Если это интерфейс длительной задачи, также сгенерируйте пример запроса статуса задачи
