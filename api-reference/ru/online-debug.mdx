## Способ использования

1. Откройте страницу целевого интерфейса и нажмите `试一试`
2. Заполните `api_key`
3. Заполните параметры пути, параметры запроса или тело запроса в соответствии с требованиями интерфейса
4. Отправьте запрос и просмотрите возвращенный результат

## Ключевые моменты отладки

- Сначала проверьте минимально жизнеспособный запрос
- Для интерфейсов длительных задач сначала проверьте `task_id` или `operation_name` в ответе о создании
- Для интерфейсов загрузки файлов необходимо убедиться, что `multipart/form-data` и имя поля файла указаны верно
- В случае ошибки сначала проверьте код состояния HTTP, `error.code` и `error.message`

## Использование совместно с кодом, сгенерированным ИИ

Рекомендуется сначала проверить минимально жизнеспособный запрос на текущей странице интерфейса, а затем использовать функцию «Копировать страницу», чтобы отправить все содержимое ИИ для генерации кода.

 Это позволит предварительно подтвердить:

- Правильность пути и метода аутентификации
- Работоспособность текущей модели параметров
- Какие идентификаторы запроса необходимо сохранить для интерфейса асинхронных задач
