## 使用方法

1. 対象のAPIドキュメントページを開き、`试一试` をクリックします
2. `api_key` を入力します
3. APIの要件に従って、パスパラメータ、クエリパラメータ、またはリクエストボディを入力します
4. リクエストを送信し、レスポンス結果を確認します

## デバッグのポイント

- まずは最小限の構成でリクエストを検証する
- 長時間実行タスクAPIでは、作成時のレスポンスに含まれる `task_id` または `operation_name` を優先的に確認する
- ファイルアップロードAPIでは、`multipart/form-data` とファイルフィールド名が正しいことを必ず確認する
- 失敗した場合は、まず HTTP ステータスコード、`error.code`、`error.message` を確認する

## AI生成コードとの連携

現在のAPIページで最小構成のリクエストを一度検証してから、「ページをコピー」機能を使用してページ全体の内容をAIに送り、コードを生成させることを推奨します。

これにより、以下の事項を事前に確認できます：

- パスと認証方式が正しいこと
- 現在のパラメータ構成で実行可能であること
- 非同期タスクAPIでどのクエリ識別子を保存する必要があるか
