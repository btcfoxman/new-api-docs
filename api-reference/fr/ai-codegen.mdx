## Pratiques recommandées

1. Ouvrez la page de l'API cible et cliquez directement sur « Copier la page ».
2. Envoyez l'intégralité du contenu copié à l'IA.
3. Indiquez à l'IA le langage de programmation, les paramètres métier et la méthode de traitement des réponses souhaités.
4. Demandez à l'IA de générer le code de requête complet, la gestion des erreurs, ainsi que la logique de requête de tâche asynchrone (si l'API implique des requêtes de tâche).
5. Revenez à la page de documentation ou de test pour vérifier la correspondance des noms de champs, des chemins et des méthodes d'authentification.

## Conseils de prompt

> Le contenu ci-dessous est le contenu intégral de la page, copié directement depuis la page de documentation API.
> Veuillez générer un code d'appel exécutable à partir de ces contenus.
> Exigences :
> 1. Utiliser l’authentification Bearer Token
> 2. Utiliser https://api.aiid.edu.kg/v1 pour Base URL
> 3. Conserver le corps de requête complet et la gestion des erreurs
> 4. Indiquer dans les commentaires les éléments à remplacer : model, clé API et paramètres métier
> 5. S'il s'agit d'une interface de tâche longue, générez également un exemple de requête de suivi de tâche
