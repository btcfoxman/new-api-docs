---

## Utilisation

1. Ouvrez la page de l'interface cible et cliquez sur `试一试`
2. Remplissez `api_key`
3. Renseignez les paramètres de chemin (path), les paramètres de requête (query) ou le corps de la requête (body) selon les spécifications de l'interface
4. Envoyez la requête et consultez les résultats retournés

## Points clés du débogage

- Validez d'abord la requête minimale viable
- Pour les interfaces de tâches longues, confirmez en priorité le `task_id` ou `operation_name` dans la réponse de création
- Pour les interfaces d'upload de fichiers, assurez-vous que le `multipart/form-data` et le nom du champ de fichier sont corrects
- En cas d'échec, vérifiez d'abord le code d'état HTTP, `error.code` et `error.message`

## Utilisation avec le code généré par l'IA

Il est recommandé de valider une requête minimale viable sur la page de l'interface actuelle avant d'utiliser « Copier la page » pour envoyer tout le contenu à l'IA pour la génération de code.

Cela permet de confirmer au préalable :

- L'exactitude du chemin et de la méthode d'authentification
- La validité du mode de paramétrage actuel
- L'identifiant de requête à conserver pour les interfaces de tâches asynchrones
