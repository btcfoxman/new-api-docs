# 创建视频

- Source: `openapi.json`
- Method: `POST`
- Path: `/v1/videos`
- Operation ID: `createVideo`

## 说明

OpenAI Sora 兼容格式的视频生成接口。支持文生视频与图/视频参考生成模式。

- `POST /v1/videos`：创建视频任务
- `GET /v1/videos/{video_id}`：查询任务状态
- `GET /v1/videos/{video_id}/content`：获取视频二进制内容（下载）

标准鉴权头：
- `Content-Type: application/json`（创建任务时）

1. 调用 `POST /v1/videos` 创建任务，拿到 `id`。
2. 轮询 `GET /v1/videos/{id}`，直到 `status=completed` 或 `failed`。
3. `completed` 后优先使用返回中的 `video_url`；如需网关统一下载可调用 `GET /v1/videos/{id}/content`。

## 3.1 核心参数

- `model: string`（必填）
- 说明：要调用的视频模型名称。
- 示例：`sora-2`、`doubao-seedance-2-0-260128`

- `prompt: string`（推荐）
- 说明：视频生成主描述。使用 `content` 时也必须传，不能为空。
- 示例：`第一人称视角果茶广告，节奏轻快`

- `seconds: number|string`（可选）
- 说明：目标时长。与 `duration` 二选一，若同时传通常以网关解析结果为准。
- 示例：`8`

- `size: string`（可选）
- 说明：输出分辨率，常用 `宽x高`。
- 示例：`720x1280`、`1280x720`

- `mode: string`（可选）
- 说明：生成模式，不传默认 `t2v`。不同模式对素材参数要求不同。
- 示例：`t2v`、`i2v`、`i2v_first_last`、`reference_material`

## 3.2 通用兼容参数

- `aspect_ratio` 或 `ratio`：宽高比。
- `duration`：时长别名（与 `seconds` 二选一即可）。
- `quality` 或 `resolution`：清晰度相关参数。
- `fps`：帧率。
- `function_mode`：可选功能模式，用于触发特定行为（如 `omni_reference` 表示全能参考语义）。

## 3.3 参考素材参数

- `input_reference`：标准参考输入对象。
- 说明：仅支持字符串 URL 写法，不支持对象写法。
- 正确示例：`"input_reference": "https://example.com/ref.png"`
- 错误示例：`"input_reference": { "image_url": "https://example.com/ref.png" }`

- `image_url` / `image_urls`：图片参考。
- 说明：`image_url` 常用于单图；`image_urls` 用于多图或全能参考。

- `end_image_url` / `last_image_url`：末帧参考。
- 说明：通常与 `mode=i2v_first_last` 搭配，表示视频结束画面。

- `video_urls`：视频参考列表。
- 说明：常用于 `reference_material` 或 `content` 全能参考场景。

- `audio_urls`：音频参考列表。
- 说明：用于背景音乐/旁白风格参考（是否生效取决于模型能力）。

- `reference_images`：多图参考。
- 说明：常用于多帧/多图参考模式。

说明：不同模型对素材字段要求不同，建议按下文“模式与最小入参”传参。

## 3.4 content 高级模式（推荐给复杂参考场景）

可直接传 `content` 数组：
- `type=text`：文本指令。
- `type=image_url|video_url|audio_url`：媒体输入。
- 支持 `name` 用于在文本中通过 `@名称` 指代素材。

必填约束：
- 使用 `content` 时，`prompt` 仍必须传且不能为空。

兼容写法：
- `image_url/video_url/audio_url` 可为字符串，或对象 `{ "url": "https://..." }`。
- `role` 可使用 `material`，也兼容 `reference_image/reference_video/reference_audio`。

可同时携带：
- `generate_audio: boolean`
- `watermark: boolean`

补充说明：
- `generate_audio=true`：请求模型同时生成音频（仅支持该能力的模型会生效）。
- `watermark=false`：请求关闭水印（实际是否生效取决于模型与渠道策略）。
- `name`：素材别名，建议与 `text` 中 `@名称` 一致，便于模型准确关联素材。

不同模型族对参数支持差异较大，建议按下表选参：

| 模型族 | 常见模型名（示例） | 推荐最小入参 | 常用扩展参数 | 注意事项 |
|---|---|---|---|---|
| Seedance | `doubao-seedance-*` | `model + prompt + mode` | `image_url/image_urls/end_image_url/reference_images/video_urls/audio_urls/function_mode/content` | 对 `mode` 最敏感；并非所有版本都支持 `reference_material`。 |
| Sora 通用 | `sora-2` `sora-2-pro` 及 `*-stable/*-official` | `model + prompt` | `seconds/size`，建议补 `input_reference`（字符串 URL） | 多渠道 fallback 时，是否要求图片可能不同，建议准备参考图提高成功率。 |
| Sora 时长预置 | `sora2-landscape-10s` `sora2-pro-portrait-25s` 等 | `model + prompt` | `image_url/input_reference` | 时长和方向通常由模型名决定，`seconds/ratio` 仅在模型允许时生效。 |
| Veo | `veo3*` `veo3.1*` `veo_*` | `model + prompt` | `mode + image_url/reference_images/content + ratio/seconds` | `i2v/r2v` 类模型通常需要参考图或参考素材；`t2v` 类通常可纯文本。 |
| Grok 视频 | `grok-video-3` | `model + prompt` | `ratio/aspect_ratio` `resolution/size` `seconds/duration` `images` | 仅走 `/v1/videos` 统一接口（创建/查询/下载），不使用渠道私有旧视频路径。 |
| Kling（官方模式） | `kling-v1` `kling-v1-6` `kling-v2-*` `kling-v3` | `model + prompt + mode` | `mode=t2v/i2v/multi_i2v`，以及 `image/image_list` 等官方字段 | 已兼容官方路径语义（text2video/image2video/multi-image2video）并统一到 `/v1/videos`；不提供视频延长对外能力。 |
| 聚合扩展模型 | `happyhorse*` `grok-imagine-1.0-video*` 等 | `model + prompt` | `seconds/size/content` | 具体支持字段依赖模型与渠道实现，建议优先使用标准字段，不足再用 `content`。 |

说明：
1. 若你使用稳定别名模型（如 `*-stable`、`*-official`），底层可能自动切换渠道，请尽量传“跨渠道兼容参数”（`prompt + model + seconds/duration + size/ratio + 必要参考图`）。
2. 复杂多素材场景（图+视频+音频+@名称）优先使用 `content`，兼容性最高。
3. 若模型文档未明确支持某字段，请按“最小入参”先跑通，再逐步增加扩展参数。

### 4.1 Kling 官方模式补充

- `mode=t2v`：文生视频
- `mode=i2v`：图生视频（需 `image` 或 `image_url`）
- `mode=multi_i2v`：多图参考（需 `image_list` 或多张 `image_urls`）
- `mode=extend`：不支持（视频延长暂不提供 `/v1/videos` 对外能力）

常见官方参数可直接放在 `/v1/videos` 请求体中透传：
- `model_name` `negative_prompt` `cfg_scale` `mode(std/pro)` `duration` `aspect_ratio`
- `sound` `camera_control` `callback_url` `external_task_id` `watermark_info`

### 4.2 Grok 视频模式补充

推荐参数：
- `model`：如 `grok-video-3`
- `prompt`
- `ratio` 或 `aspect_ratio`
- `resolution` 或 `size`
- `seconds` 或 `duration`
- `images`（可选，单图参考时建议传 1 张）

- `mode=t2v`
- 最小入参：`model + prompt`

- `mode=i2v`（等同 `i2v_first`）
- 最小入参：`model + prompt + image_url`（或 `input_reference` 字符串 URL）

- `mode=i2v_first_last`
- 最小入参：`model + prompt + image_url + end_image_url`

- `mode=reference_images`
- 最小入参：`model + prompt + reference_images`（通常 1~4 张）

- `mode=reference_material`
- 最小入参：`model + prompt + (image_urls/video_urls/audio_urls 至少一种)`
- 复杂场景更推荐用 `content + function_mode=omni_reference`
- 说明：当传 `function_mode=omni_reference` 时，即使未显式传 `mode=reference_material`，也会按全能参考语义处理。
