title: "状态码与排障"

---

new-api 遵循标准的 HTTP 状态码规范，并对 AI 场景做了特定的业务语义增强。

## 常见状态码映射表

| 状态码 | 含义 | 建议解决方案 |
| :--- | :--- | :--- |
| **200** | 请求成功 | - |
| **400** | 无效请求 | 检查 `model` 字段是否正确，或 Prompt 是否包含违规词。 |
| **401** | 鉴权失败 | 令牌 (sk-) 错误、过期或 Base URL 配置不当。 |
| **403** | 权限拒绝 | 您尝试访问的模型不在当前令牌的权限范围内。 |
| **404** | 路径不存在 | 请确保 Base URL 以 `/v1` 结尾。 |
| **429** | 触发限流 | 您的请求频率过高，或当前账号额度已耗尽。 |
| **500** | 内部服务错误 | 上游服务商 (如 OpenAI) 异常，通常重试即可解决。 |
| **503** | 服务不可用 | 模型当前处于维护状态或负载过高。 |

## 调试建议

如果您在使用过程中遇到持续的错误，请：
1. **检查控制台日志**：在 new-api 控制台查看该次请求的具体失败原因。
2. **验证网络连通性**：使用 `ping` 或 `curl` 测试 API 服务器的连通性。
3. **分段排查**：先在终端用 `curl` 尝试，排除第三方工具配置问题的可能性。
