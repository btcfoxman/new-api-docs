## Token Authentication

Chuhaiying API uses standard Bearer Token mechanisms for authentication.

### Header Format
```text
Authorization: Bearer YOUR_API_KEY
```

## Token Security Standards

### 1. Principle of Least Privilege
If you distribute a API Key to external partners, please use the console to restrict the following for that key:
- **Model Scope**: Only allow calls to specific models.
- **Expiration Time**: Set short-term expiration to reduce risk.
- **IP Whitelist**: Restrict the IP addresses of the caller.

### 2. No Client-Side Hardcoding
Never hardcode `sk-` tokens directly into frontend JavaScript or mobile app code. All API calls should be proxied via a backend server or use short-term tokens generated through our secure distribution system.

### 3. Data Privacy
We do not store your conversation content. API acts solely as a high-performance relay node; conversation content flows through memory and is passed directly to the underlying service providers.

## Monitoring and Auditing

You can monitor the details of every API call in real-time on the "Logs" page of the console. For security concerns, please email support@aiid.edu.kg.
