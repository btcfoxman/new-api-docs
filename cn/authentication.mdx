description: "API Key 的使用规范与安全说明"
---

## 认证方式

所有 API 请求都必须在 HTTP Header 中包含您的 API Key。

### HTTP Header 示例

```text
Authorization: Bearer sk-your-api-key
```

## 安全建议

1. **环境变量**：切勿将 API Key 硬编码在代码中。建议使用 `.env` 文件或环境变量。
2. **控制台管理**：您可以随时在控制台撤回、轮换或限制特定 API Key 的配额。
3. **分发策略**：为不同的项目或团队成员生成独立的 Key，以便精准监控使用情况。

## 错误代码

如果鉴权失败，API 将返回 `401 Unauthorized`。请检查 Key 是否有效、是否已启用或是否配置了正确的 Base URL。
