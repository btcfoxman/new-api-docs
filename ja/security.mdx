## トークン認証 (Authentication)

出海营 API は、標準的な Bearer Token メカニズムを使用して本人確認を行います。

### Header 形式
```text
Authorization: Bearer YOUR_API_KEY
```

## トークンセキュリティ規範

### 1. 最小権限の原則
API キーを外部パートナーに配布する場合は、コンソールでそのキーに対して以下の制限を設定してください。
- **モデル範囲**：特定のモデルの呼び出しのみを許可します。
- **有効期限**：リスクを軽減するため、短期間で失効するように設定します。
- **IP ホワイトリスト**：呼び出し元の IP アドレスを制限します。

### 2. クライアント側でのハードコーディングの厳禁
フロントエンドの JavaScript やモバイルアプリのコードに `sk-` トークンを直接書き込まないでください。すべての API 呼び出しはバックエンドサーバーを経由するか、当社の安全な配信システムを通じて短期間のトークンを生成する必要があります。

### 3. データプライバシー
当社はお客様の対話内容を保存しません。API は高性能な中継ノードとしてのみ機能し、対話内容はメモリ内を流動し、基盤となるサービスプロバイダーに直接透過的に転送されます。

## モニタリングと監査

コンソールの「ログ」ページで、各 API 呼び出しの詳細をリアルタイムで監視できます。セキュリティに関するご質問は、support@aiid.edu.kg までメールでお問い合わせください。
